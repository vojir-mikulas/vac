security posture is genuinely good тАФ **no XSS, no `dangerouslySetInnerHTML` on user data, no

The findings below are refinements, not fires. Suggested fix order: the two Med bugs (CopyButton
timer, blob-URL revoke) тЖТ the step-up gate items тЖТ quality polish.

## Security

Mostly clean. Notes:

- **CSRF double-submit is correct** (`lib/api/client.ts:84-87`) тАФ header sent only on unsafe verbs,
  cookie read JS-side. By design the `vac_csrf` cookie is JS-readable; that's correct for
  double-submit, but it means an XSS would defeat CSRF. Since everything is escaped today, this is
  acceptable тАФ keep it that way.
- **Log rendering is safe** тАФ log `message`/`service` render as JSX text children, never raw HTML.
  `chart.tsx`'s `dangerouslySetInnerHTML` injects only developer chart config, not user data.
- **LOW тАФ Generated DB secrets shown via toast** (`features/addons/addons-page.tsx`, install dialog):
  `generated_secrets` are concatenated into a Sonner toast string. "Shown once" by design, but a toast
  has no copy affordance and a short lifetime, nudging the operator to screenshot. Prefer the masked
  read-only input + `CopyButton` pattern used in `auto-deploy-section.tsx`.

## Bugs

### ЁЯЯа MED тАФ `CopyButton` leaks a timer / can setState after unmount
`components/common/copy-button.tsx:48`
```ts
setTimeout(() => setCopied(false), 1500)   // never cleared
```
CopyButton lives in lists and dialogs that often unmount within 1.5s. Store the id in a ref and clear
it in a `useEffect` cleanup (and before re-scheduling).

### ЁЯЯа MED тАФ `downloadFile` revokes the blob URL synchronously and uses a detached anchor
`lib/log-export.ts:29-33`
```ts
a.click()
URL.revokeObjectURL(url)   // can cancel the download in Firefox/Safari
```
The anchor is never appended to the DOM, and the revoke races the download. Fix: append to
`document.body`, click, remove, and defer the revoke (`setTimeout(() => URL.revokeObjectURL(url), 0)`).

### ЁЯЯб LOW тАФ Step-up submit gate assumes 6-digit TOTP even for recovery codes
`components/auth/step-up-provider.tsx:135`
```ts
disabled={submitting || code.length < 6}
```
Recovery codes aren't necessarily тЙе6 chars in that format. Gate the length rule with `!useRecovery`.

### ЁЯЯб LOW тАФ Step-up form can double-submit via Enter
`components/auth/step-up-provider.tsx:88-93` тАФ the submit `<Button>` is disabled while `submitting`,
but pressing Enter still fires `onSubmit` тЖТ `submit()`, which has no guard. Add `if (submitting) return`
at the top of `submit()`.

### ЁЯЯб LOW (VERIFY) тАФ Container-shell resize can hit a disposed terminal
`features/app-detail/use-container-shell.ts:85-101` тАФ the resize listener (`fitRef.current?.fit()`)
and the dispose-on-unmount live in separate effects, so a resize firing during teardown could call
`fit()` on a disposed terminal. Null `fitRef.current` in the unmount cleanup, or guard `onResize` on
`termRef.current`. (Confirmed the original "status stuck on connecting" concern is a non-issue:
`socket.onclose` always sets `'disconnected'`.)

## Code quality

- **LOW тАФ `features/logs/log-explorer.tsx` ships hardcoded English strings** ("Logs", "Select an
  appтАж", etc.) while every other feature uses `useTranslation`. Inconsistent and breaks localization;
  `eslint-plugin-i18next` is configured but this slipped through.
- **LOW тАФ `features/deployments/deployments-page.tsx`**: the `useQueries` fan-out has no `staleTime`
  (N refetches per visit on instances with many apps), and `sort()` + `computeMetrics()` run inline on
  every render. Wrap the merge/sort/metrics in `useMemo`, add `staleTime`.
- **LOW тАФ Fragile remount-to-reseed** (`features/settings/notifications-section.tsx`):
  `key={JSON.stringify(data.events)}` discards unsaved Discord/Slack input on any query revalidation.
  Prefer the explicit "track server value in state" pattern used in `env-tab.tsx`.
- **LOW тАФ `features/settings/totp-section.tsx`**: the dialog open button isn't wrapped in
  `DialogTrigger`, so Radix focus-return on close is bypassed тАФ inconsistent with the other dialogs.
- **NIT** тАФ `apps-dashboard.tsx` `useMemo(() => apps ?? [], [apps])` is pointless; module-level
  `nextUid` in `env-tab.tsx` grows unbounded per session (harmless).
